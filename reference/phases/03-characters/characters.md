# 阶段 03：人物工坊

## 触发条件

用户说"设计角色""新人物""这个角色怎么"时加载本手册。

## 流程

### 1. 确认角色功能

服务于故事，非反过来：角色在核心冲突中的位置？对主角弧线的作用？

### 2. 设计 Lie + Want + Need

- **Lie**（错误信念）：驱动错误决策的根本认知偏差
- **Want**（想要）：Lie 驱动下角色自己知道的目标
- **Need**（需要）：故事要教给角色的真相，往往与 Lie 相反

### 3. 设计声线

遮住名字也能分辨，至少选 3 项：句长偏好 / 习惯词尾音 / 敬语程度 / 说话节奏 / 方言口吻。

### 4. 设计关系网

与已有角色标注关系类型和温度（1-10）。1-3 敌对，4-6 中立，7-8 友好，9-10 羁绊。

### 5. 设计成长弧线

起点（登场状态）→ 转折点（什么打破 Lie）→ 终点（故事结束时的状态）。无弧线配角标注核心功能和记忆标签。

### 6. 录入台账

写入 `tracking/character-status.md`（如果不存在则先用 `templates/character-status.md` 模版创建）。

---

## 完成标准

- [ ] 故事功能已确认
- [ ] Lie + Want + Need 完整自洽
- [ ] 声线 ≥ 3 要素
- [ ] 与主角关系已标注
- [ ] 弧线或记忆标签已定义
- [ ] `tracking/character-status.md` 已录入

---

## 常见错误

1. 先设计角色后想功能
2. Want 和 Need 写反
3. Lie 是性格小缺点而非根本认知错误
4. 所有角色同一种声线
5. 配角只有名字种族没有记忆标签
6. 恋爱角色没有独立弧线（工具人）
