# 阶段 05：初稿写作

## 触发条件

用户说"写第X章""继续写""下一章"时加载本手册。

## 写前准备

**必须完成以下所有检查项后才能开始写正文。**

### 0. 检查未定稿章节

扫描 `workspace/novels/{name}/content/` 中是否存在 `chXX-draft.md` 文件。如有，告知用户："以下章节尚未定稿：[列出]。建议先定稿再开新章，避免台账数据过期。"

### 1. 读取大纲确认本章定位

读取`workspace/novels/{name}/outline/`和前两章小说正文内容，确认本章在大纲中的位置和功能：

- 本章属于哪一卷/哪一段弧线？
- 大纲给这章分配了什么功能（推进主线 / 展示人物 / 埋设伏笔 / 过渡 / 高潮准备）？
- 与前章的衔接点、向后章的钩子方向

如果 outline.md 不存在或本章未在大纲中覆盖，则告知用户并建议补充大纲。如果用户仍要求继续写，则从 tracking/progress.md 和已写章节中推导本章方向。

### 2. 检查角色台账

读取 `workspace/stories/{name}/tracking/character-status.md`。

确认本章涉及角色的当前状态：
- 是否有台账中不存在的新角色登场，如有，则需要先加载`characters.md`完成角色设计
- 上次登场章节和状态（受伤？疲劳？情绪？）
- 当前能力/阶位/装备
- 当前人际关系温度
- 未完成的个人弧线节点

### 3. 检查进度文档

读取 `workspace/stories/{name}/tracking/progress.md`。确认：
- 当前时间线（本章日期）
- 已埋未收的伏笔（本章是否该收？）
- 开放线索（本章是否该推进？）
- 上一章的结尾状态（本章第一拍必须接住）

### 4. 预读相关设定

根据本章定位和大纲，从权威文档中预读本章可能触及的设定段落，刷新记忆：

- `brief.md`：明确小说基本设定及要求
- `world.md`：本章涉及的种族/地理/魔法/势力设定
- `outline.md`：本章在大纲中的位置、前后章衔接点
- `rules.md`：项目风格规范、特殊约束

**目的**：让大脑在动笔前装好"本章要用到的设定"。具体数值/归属/描述是否用对——那是写完后再核验的事（见写后自查第 7 项）。

如果无从判断"本章可能触及什么"（目标模糊或对设定不熟），先通读 `world.md` 和 `outline.md` 中与当前进度相关的段落，再做本章目标确认（回到步骤 1）。

### 5. 检查外部系统

读取 `workspace/stories/{name}/rules.md`，检查是否配置了外部记忆/追踪系统。如有，在写作前按 `rules.md` 中的指令查询相关信息（如语义召回当前进度、角色状态、最近事件等）。

### 6. 检查章首格式

确认项目 `rules.md` 中定义的章首格式（日期标注等），准备好元数据。

### 7. 可用工具及模块

`consistency_review`模块，用于去 AI 味，在写初稿前加载。

## 章节起草前声明

输出正文前，先按照以下格式输出章节起草前声明，如果不清楚或不确定，则提醒用户哪些信息缺失，并给出解决建议。

```
─── 第X章 起草声明 ───
目标：[本章要推进/揭示/转折什么]
POV：[视角角色]
节拍：1.[事件] 2.[事件] 3.[事件]
情感：[起点情绪] → [终点情绪]
约束：[必须遵守的已确立事实]
───────────────────────
```

## 正文执行

### 情况 A：从零写（章节文件为空或不存在）

1. 用 `write_file` 写入 `workspace/novels/{name}/content/chXX-draft.md`
2. 写作中遵循 `workspace/novels/{name}/rules.md` 的风格规范（如存在）
3. 写完立即进入「写后自查」

### 情况 B：已有初稿（章节文件非空，用户要求修订）

1. 先过「写后自查」全部检查项，定位问题
2. 补充「写前准备」中可能遗漏的事实核验
3. 用 `write_file` 一次性改写全章（不用多次 `patch`）
4. 改写后再过一遍「写后自查」

**执行正文写作时遵守**：
- 保持 Brief 中确定的文风一致性
- POV 不乱切（除非设计了切换）
- 对话有辨识度（不同角色说话方式不同）
- 展示 > 叙述（Show don't tell）
- 节奏控制：紧张段落短句多，舒缓段落长句多

## 写后自查

写完正文后，立即逐项检查。**不过不交付。**

### 1. 剧情逻辑

- 本章关键事件的前提、触发、后果是否完整？
- 事件因果关系有无跳跃？（从 A 直接到 C 缺了 B）
- 同一操作出现两次结果不同时，是否已显式区分条件？

### 2. 目标一致性

- 主角本章的行为是否与当前总体目标一致？
- 如有偏离，是否是有意的（展示另一面/被迫选择）？

### 3. 情绪与关系

- 情绪变化是否有来路？（不能凭空愤怒/感动）
- 人物间关系温度是否承接前章？（不能忽冷忽热）

### 4. 身体与状态信息

- 角色的伤势/疲劳/身体限制是否被持续追踪？
- 重要道具/装备的状态是否一致？

### 5. 场景转场

- 场景切换时读者会不会迷路？（至少给出一个锚点：地点/时间/人物）
- 连续场景间是否有呼吸感？（紧张段落接松弛段落）

### 6. 章末承接

- 章末收在变化上还是画面上？（变化 > 画面 > 总结）
- 下一章第一拍能否自然接住本章结尾？

### 7. 事实一致性

- 本章所有涉及已建立事实的地方，与 `world.md` + `outline.md` + `rules.md` 是否一致？
- 如有新增设定，是否与已有设定冲突？

### 8. 常识逻辑

- 章节内的基本因果关系是否自洽？
- 时间顺序是否合理？（不能先发生后触发）
