# 阶段 04：世界构建

## 触发条件

用户说"设定""世界观""魔法系统""规则有问题"时加载本手册。

## 原则

1. **设定服务故事** — 从核心冲突倒推世界需要什么，只在故事需要时才设计
2. **规则 > 描述** — 能做什么/不能做什么 > 长什么样
3. **边界比内容重要** — 什么不能发生比什么都更关键
4. **如果用户已有世界设定则检查缺少的部分并提醒** — 不要假装不知道并从头开始设计

## 流程

### 1. 从故事需求倒推

读 `brief.md` 。从核心冲突出发：
- 冲突需要什么舞台？→ 设计势力/地理
- 主角特殊需要什么规则？→ 设计能力体系
- 情感引擎需要什么氛围？→ 设计暗面规则

### 2. 设计规则体系

魔法/力量体系：来源 → 获取 → 代价 → 阶位。关键：最强者和最弱者差距多大？社会代价是什么？

社会/政治体系：权力分配 → 阶层流动 → 法律禁忌。主角阶层带来什么限制/特权？

### 3. 设计历史锚点

只设计直接影响当前故事的事件。每条标"多久前 + 谁还活着"（亲历者视角 > 历史书）。

### 4. 标注关键数值

技能参数、种族特征、地理尺度、阶层比例。数值一旦写入后文不得翻供。

### 5. 标注禁忌与边界

至少 3 条"绝对不行"。魔法禁忌 / 社会禁忌 / 物理限制。边界让世界有形状。

### 6. 写入

写入 `world.md`（用 `templates/world.md` 模版）。

---

## 完成标准

- [ ] 力量体系：来源/获取/代价/阶位已定义（如适用）
- [ ] 社会体系：权力/阶层/禁忌已定义
- [ ] 关键历史事件已录入（直接影响当前故事）
- [ ] 会被引用的数值已确立
- [ ] 禁忌与边界 ≥ 3 条
- [ ] `world.md` 已写入

---

## 常见错误

1. 堆设定不服务故事（12 种族只用 3 个 = 9 条噪音）
2. 只有描述没有规则（"很强大"没有参数）
3. 数值想当然不写入文档（凭记忆 = 前后矛盾）
4. 没有禁忌（什么都能做 = 没紧张感）
5. 历史成百科全书（写亲历者视角不写编年史）
6. 为"酷"牺牲一致性（破坏读者对世界规则的信任）
