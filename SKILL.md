# Hermes Novel Studio

全流程网文创作工作室——从灵感到完本，一站式完成。

## 快速决策

| 用户意图 | 执行流程 |
|----------|----------|
| "帮我写一部小说" | 执行 **完整流程**（构思→大纲→写作→导出） |
| "续写第N章" | 读取项目状态 → 直接进入 **写作阶段** |
| "给主角生成一张图" | 加载 `reference/image-prompts.md` → 生成角色图 |
| "审查我的小说" | 运行 `continuity.py` + LLM 审查 → 输出报告 |
| "导出全本" | 运行 `export.py` → 交付手稿 |
| “调整大纲/重新规划” | 进入 **大纲阶段**，保留已写章节 |

---

## 完整流程

```
用户请求 → [1.构思] → [2.大纲] → [3.写作] → [4.修订] → [5.导出]
                ↕                        ↕
           [图片生成] ←── 穿插 ──→ [图片生成]
```

## 阶段 0：项目初始化

首次创作时，运行 `scripts/init_project.py` 创建项目骨架：

```bash
python3 scripts/init_project.py "书名" --output ./novels
```

这会创建完整目录：
```
novels/书名/
├── meta.json           # 项目元数据
├── outline.md          # 大纲
├── worldbuilding.md    # 世界观设定
├── characters/         # 人物卡
├── chapters/           # 章节正文
├── images/             # 配图
└── export/             # 导出产物
```

如果项目已存在（续写/恢复），跳过此阶段，直接读取 `meta.json` 恢复状态。

## 阶段 1：构思（交互式需求采集）

**逐一询问用户**，每次 1-2 个问题，不要一次性抛所有问题。

### 第一轮：核心定位
问用户：
1. **小说类型**：玄幻 / 都市 / 悬疑 / 科幻 / 历史 / 仙侠 / 言情 / 游戏电竞 / 其他
2. **风格定位**：热血爽文 / 轻松搞笑 / 深沉黑暗 / 悬疑烧脑 / 情感治愈 / 脑洞大开

### 第二轮：规模与平台
3. **目标字数**：5万字 / 10万字 / 20万字 / 50万字 / 自定义
4. **目标平台**（可选）：番茄小说 / 起点中文网 / 晋江 / 通用（不限定）

### 第三轮：核心创意
5. **一句话卖点/核心创意**（用户提供，或由 Agent 根据类型+风格自动生成 3 个方案供选择）
6. **主角设定偏好**（用户提供，或 Agent 自动生成 2 个方案供选择）

### 构思输出
构思完成后，将结果写入 `meta.json` 并口头总结给用户确认：
```
✅ 构思确认：
   📖 类型：都市重生
   🎨 风格：热血爽文 + 商战权谋
   📊 目标：50万字（约 160 章）
   🏷️ 卖点：商界大佬重生1998，先知先觉打造万亿帝国
   👤 主角：林风，35岁重生，前世顶级投资人，今生从大学开始布局
```

**确认后进入大纲阶段。**

## 阶段 2：大纲与设定

**加载参考文档**（按需）：
- 选择类型 → 读取 `reference/genre-guides.md` 对应类型章节
- 用户指定平台 → 读取 `reference/platform-tactics.md` 对应平台章节

### 2.1 世界观构建
根据类型，自动生成世界观设定，写入 `worldbuilding.md`。必须包含：
- 世界背景（时代/地理/规则）
- 力量/等级体系（如适用）
- 势力分布
- 核心规则/设定

### 2.2 人物设定
为每个主要角色创建人物卡，使用 `templates/character-sheet.md` 模板，保存到 `characters/` 目录：
- 主角（必含核心欲望、金手指/能力、性格弧线）
- 核心配角（3-5 个，含与主角的关系）
- 主要反派（1-2 个，含动机/目标）

### 2.3 章节大纲
生成完整章节规划，使用 `templates/project/outline.md` 模板，写入 `outline.md`。三幕结构：
- **第一幕（10-15%）**：建立世界观、引入主角、首个冲突
- **第二幕（70%）**：成长线+冲突升级+支线展开
- **第三幕（15-20%）**：高潮+收束+结局

章节规划表至少包含每章的：标题、核心剧情点、目标字数。

### 2.4 用户确认
展示完整大纲后询问：
- "大纲是否满意？可以：① 确认开始写作 ② 局部修改 ③ 重新生成"

**大纲必须用户确认后才进入写作阶段。**

## 阶段 3：写作（逐章生成）

### 写作前准备
加载以下上下文：
- `outline.md` 当前章节的剧情点
- 前 3 章的完整内容（保持语气、节奏连贯）
- `characters/` 中本章涉及人物的人物卡
- `worldbuilding.md` 中本章涉及的设定
- 加载 `reference/writing-techniques.md` 作为写作指导

### 每章生成规范
使用 `templates/chapter.md` 模板：
1. **章首钩子**（前 3 句抓住注意力）
2. **正文**（目标字数 ±20%）
3. **章末悬念**（信息差 / 抉择 / 危机 / 反转 四选一）
4. **本章自检清单**

章节文件命名：`chapters/{章节编号}-{章节标题}.md`，如 `chapters/001-重返1998.md`

### 每章完成后
- 更新 `outline.md` 中该章状态为 ✅
- 更新 `meta.json` 中 `completed_chapters` 和 `current_words`
- 记录新增人物/地点/伏笔到对应文件
- 运行 `scripts/stats.py` 展示进度

### 写作节奏指导
参考 `reference/writing-techniques.md`：
- 黄金三章法则（前 3 章必须钩住读者）
- 爽点分布（3-5 章一小爽，10-15 章一大爽）
- 章尾必须有钩子（信息差/抉择/危机/反转）

### 断点续写
如果写作中断（用户离开 / 上下文溢出）：
1. 当前进度全部保存在文件中（`meta.json` + 章节文件）
2. 下一个会话只需读取 `meta.json` 即可恢复
3. 从 `completed_chapters` 下一章开始续写

## 阶段 4：图片生成（穿插进行）

### 何时生成图片
- 主要角色创建后 → 生成角色图
- 重要场景首次出现 → 生成场景图
- 写作完成后 → 生成封面图
- 用户主动要求 → 随时生成

### 生成流程
1. 加载 `reference/image-prompts.md` 获取 prompt 模板
2. 根据人物卡/场景描述，填充模板生成 prompt
3. 使用图片生成工具（用户配置的模型）生成
4. 图片保存到 `images/` 目录
5. 文件名格式：`角色名.png` / `scene-{编号}.png` / `cover.png`

## 阶段 5：质量审查与修订

### 自动检查
运行 `scripts/continuity.py <项目目录>` 进行结构化检查：
- 章节序号连续性
- 人物是否有人物卡
- 大纲覆盖情况
- `[NEED_LLM]` 标记项（需要 Agent 深入检查）

### LLM 深度审查
针对 `continuity.py` 标记的 `[NEED_LLM]` 项，Agent 执行：
1. 人物一致性：抽查 3-5 章，检查人物性格/能力/关系是否与人物卡一致
2. 设定一致性：检查世界观设定是否前后矛盾
3. 节奏检查：评估爽点分布是否合理，是否有"水文"章节
4. 伏笔追踪：检查已埋伏笔是否有回收计划

输出格式：
```
📋 质量审查报告

🔴 硬伤（必须修改）：
  - 第12章：主角能力描述与第3章设定矛盾
  - 第25章：人物A的性格突变，缺乏过渡

🟡 建议优化：
  - 第8-10章节奏偏慢，建议合并或加速
  - 第15章节缺少章末钩子

🟢 亮点：
  - 第1-3章黄金三章效果出色
  - 反派人设立体，动机可信
```

## 阶段 6：导出

运行 `scripts/export.py <项目目录>`：
```bash
# 标准导出（含目录页、章节标题）
python3 scripts/export.py novels/书名/

# 纯正文模式
python3 scripts/export.py novels/书名/ --no-chapter-titles

# 导出纯文本
python3 scripts/export.py novels/书名/ --format txt
```

导出产物在 `export/` 目录，告知用户文件路径。

## 重要约束

1. **所有内容写入文件**：大纲、人物卡、章节都必须落盘，绝不只存于对话中
2. **按顺序执行**：不跳过阶段，每个阶段用户确认后进入下一阶段
3. **保持连续性**：写作时始终参考前 3 章内容和人物卡
4. **用户拥有最终决定权**：大纲、人物设定、重要情节转折必须用户确认
5. **脚本优先**：能用脚本完成的操作（初始化/统计/导出/检查）优先用脚本，保证确定性
6. **项目目录不可变**：所有小说项目统一放在 `./novels/` 下，不散落各处
